# Quickstart: Lobby Visual Foundation

**Feature**: 019-lobby-visual-foundation
**Date**: 2026-04-16

## Prerequisites

```bash
pnpm quickstart    # Supabase + migrations + seed + .env.local (idempotent)
pnpm dev           # Start Next.js dev server on http://localhost:3000
```

At least 3 player accounts help validate the directory (cap behaviour needs 25+ to test "Show all"; use the seed script or log in through three browser profiles).

## Development Workflow

### 1. Baseline (zero-warnings before any change)

```bash
pnpm lint
pnpm typecheck
pnpm test           # Unit + contract
pnpm test:integration
```

If any of these fail before the feature work begins, resolve first. The feature enters on a clean tree.

### 2. TDD cycle per task

Each task in `tasks.md` (generated by `/speckit.tasks`) follows the Red → Green → Refactor rhythm. Representative cycle for a new primitive:

```bash
# Red — write failing test first
pnpm test:unit -- tests/unit/components/ui/Button.spec.tsx

# Green — implement minimum code until the spec passes
# (edit components/ui/Button.tsx)

# Refactor — keep tests green, apply Clean Code
pnpm test:unit -- tests/unit/components/ui/Button.spec.tsx

# Commit each passing test separately
git add tests/unit/components/ui/Button.spec.tsx components/ui/Button.tsx
git commit -m "feat(ui): add Button primitive with variant/size contract"
```

### 3. Key test commands

```bash
# All new primitive specs
pnpm test:unit -- tests/unit/components/ui/

# Pure helpers
pnpm test:unit -- tests/unit/lib/ui/avatarGradient.spec.ts
pnpm test:unit -- tests/unit/lib/lobby/directoryOrdering.spec.ts
pnpm test:unit -- tests/unit/lib/lobby/heroWords.spec.ts

# Lobby component specs
pnpm test:unit -- tests/unit/components/lobby/

# E2E — lobby visual + presence
pnpm exec playwright test tests/integration/ui/lobby-visual.spec.ts
pnpm exec playwright test tests/integration/ui/lobby-presence.spec.ts

# Full regression before PR
pnpm lint && pnpm typecheck && pnpm test && pnpm test:integration
pnpm exec playwright test
```

### 4. Visual / UX verification

```bash
pnpm dev
```

- Open http://localhost:3000 logged out → confirm hero, ORÐUSTA subtitle, rotating Icelandic words every ~5 s.
- Log in; confirm Play Now dominates the fold, Ranked pre-selected, Casual/Challenge disabled with tooltip.
- Open a second browser profile, log in, open invite dialog from card Challenge action, accept from window 2, navigate into match together.
- DevTools → rendering → emulate **reduced motion** → reload; confirm no keyframes play, word cycle halts on a single fixed word, state changes remain observable.
- DevTools → device toolbar → iPhone 12 (390×844) → confirm single column, sticky Play Now, bottom-sheet invite dialog, no horizontal scroll.
- DevTools → Lighthouse (mobile, throttled 4G) → run against `/` → confirm Performance ≥ 90, LCP < 2.0 s, CLS < 0.05.
- DevTools → Accessibility tree / axe extension → confirm zero serious/critical violations on logged-out and logged-in routes.

### 5. Keyboard / screen-reader sweep

- macOS Safari + VoiceOver (`Cmd+F5`): tab from the top, confirm logical order, confirm `role="dialog"` announcement when invite opens, confirm Escape returns focus.
- Windows Firefox + NVDA: same flow.
- Status changes (invite arrival, queue match) announced via `aria-live`; focus never leaves the user's current context.

### 6. Performance marks

Open DevTools → Performance → record a reload on `/`:

- `lobby:lcp-candidate` fires at hero first paint.
- `lobby:first-interaction` fires on first focus inside `PlayNowCard`.

Spikes or missing marks indicate regression work before shipping.
